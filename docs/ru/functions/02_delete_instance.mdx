# Удаление экземпляра

:::warning
**Важные замечания перед удалением**

- Эта операция является **безвозвратной** - удаленные экземпляры не могут быть восстановлены
- Убедитесь, что у вас есть [действительные резервные копии](../functions/15_back_restore.mdx), если вам могут понадобиться данные позже
- Убедитесь, что никакие приложения в настоящее время не используют этот экземпляр базы данных
- Удаление кластера приведет к удалению всех реплик и связанных ресурсов
  :::

Удаление экземпляра PostgreSQL приведет к удалению:

- Пользовательского ресурса (CR)
- StatefulSet и Pods
- ConfigMaps и Secrets
- PersistentVolumeClaims (PVC)
- Ресурсов мониторинга и журнала

## Процедура

<Tabs>
  <Tab label="CLI">
    Чтобы удалить экземпляр PostgreSQL (включая все связанные PVC), выполните:

    ```bash
    kubectl -n $NAMESPACE delete postgresql <instance-name>
    ```

    Пример вывода:

    ```bash
    postgresql.acid.zalan.do "example-postgres" удален
    ```

    После удаления вы можете подтвердить статус экземпляра с помощью:

    ```bash
    kubectl -n $NAMESPACE get postgresql
    ```
  </Tab>

  <Tab label="Веб-консоль">
    1. Перейдите в раздел **PostgreSQL** в левом меню

    2. Выберите целевое пространство имен.

    3. Выберите экземпляр, который вы хотите удалить, из списка экземпляров.

    4. На странице деталей экземпляра нажмите кнопку **Действия** в правом верхнем углу.

    5. Выберите опцию **Удалить**.

    6. Подтвердите удаление экземпляра и всех его данных.

    7. Введите имя экземпляра в окне подтверждения.

    8. Нажмите **Удалить**.
  </Tab>
</Tabs>

## Проверка после удаления

После удаления проверьте, что все ресурсы удалены:

```bash
kubectl -n default get postgresql  # Не должен отображать удаленный экземпляр
kubectl -n $NAMESPACE get pvc  # Проверьте, что PVC были удалены
```
