# Журнал

## Введение

PostgreSQL предоставляет надежную систему ведения журналов, включая подробные журналы ошибок. Понимание этих журналов, которые фиксируют события во время работы экземпляра на уровне контейнера в вашем кластере Kubernetes, является критически важным. Правильный анализ журналов, особенно журналов ошибок, помогает вам быстро определить коренную причину проблем, эффективно устранять их и быстро реагировать на сбои или исключения.

## Процедура

<Tabs>
  <Tab label="CLI">
    1. Просмотрите все связанные поды экземпляра. Замените `${namespace}` на фактическое пространство имен, а `${instance_name}` на имя экземпляра. Используйте соответствующий селектор меток для вашего экземпляра PostgreSQL.

    ```bash
    # Пример с использованием стандартного селектора меток Alauda
    kubectl get pod -n ${namespace} -l "middleware.instance/name=${instance_name}"
    ```

    2. Запросите журналы для конкретного контейнера `postgres` пода. Замените `${namespace}` и `${pod_name}` на фактические значения.

    ```bash
    kubectl logs -n ${namespace} ${pod_name} -c postgres
    ```
  </Tab>

  <Tab label="Web Console">
    1. В левой панели навигации нажмите **PostgreSQL**.
    2. Нажмите ***Имя пространства имен***.
    3. Нажмите ***Имя экземпляра***.
    4. В закладке **Журналы** установите параметр просмотра для отображения соответствующего журнала.
  </Tab>
</Tabs>

**Примечание:** По умолчанию журналы PostgreSQL внутри контейнера обычно хранятся в каталоге `/var/log/pg_log`, часто в формате `.log` или `.csv`. Точная путь и формат могут зависеть от конфигурации PostgreSQL.
