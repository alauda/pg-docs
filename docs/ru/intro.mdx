# Введение

## Введение в PostgreSQL

**PostgreSQL** — это мощная, open-source объектно-реляционная система управления базами данных, известная своей надежностью, богатством функционала и производительностью. Она использует и расширяет язык SQL, дополненный многими функциями для безопасного хранения и масштабирования самых сложных операций с данными. PostgreSQL работает на всех основных операционных системах и соответствует требованиям ACID с 2001 года. Широко используется для различных приложений — от одиночных машин до хранилищ данных и веб-сервисов с множеством одновременных пользователей.

## Поддержка PostgreSQL от Alauda

**Поддержка PostgreSQL от Alauda** — это встроенное в Kubernetes решение для управления, основанное на надежной основе оператора Zalando Postgres. Оно упрощает развертывание и полное управление жизненным циклом кластеров PostgreSQL в средах Kubernetes с использованием определений пользовательских ресурсов (CRD). Эта служба автоматизирует сложные операционные задачи, включая настройку высокой доступности, автоматическое переключение на резервирование, масштабирование, поэтапные обновления и резервное копирование/восстановление, позволяя предприятиям без усилий запускать базы данных PostgreSQL производственного уровня на Kubernetes.

## Ключевые особенности

Ключевые особенности, предлагаемые поддержкой PostgreSQL от Alauda, включают:

- **Автоматизированные операции**: Использует шаблон оператора для автоматического управления развертыванием кластеров, изменениями конфигурации, масштабированием и задачами резервного копирования/восстановления, значительно уменьшая ручные усилия и потенциальные ошибки.

- **Высокая доступность (HA)**: Интегрирует Patroni для надежных механизмов автоматического переключения на резервирование, выборов лидера и мониторинга состояния кластера, обеспечивая непрерывную доступность службы базы данных.

- **Эластичное масштабирование**: Позволяет динамическое горизонтальное масштабирование (добавление/удаление реплик) и вертикальное масштабирование (корректировка ресурсов CPU/памяти) для адаптации к изменяющимся нагрузкам без простоя.

- **Резервное копирование и восстановление**: Поддерживает запланированные и ручные резервные копии на совместимое с S3 хранилище (AWS S3, Google Cloud Storage, Azure Blob Storage, MinIO), обеспечивая надежную защиту данных и восстановление в точке времени (PITR).

- **Безопасностные функции**: Обеспечивает параметры безопасности уровня предприятия, включая шифрование TLS для данных в пути, интеграцию Kubernetes RBAC для контроля доступа и поддержку безопасного управления паролями.

- **Мониторинг и ведение журналов**: Предлагает встроенную интеграцию с Prometheus для сбора метрик и обеспечивает легкий доступ к журналам PostgreSQL, позволяя получить полное представление о работе и состоянии базы данных.

- **Пользовательская конфигурация**: Позволяет широкую настройку параметров PostgreSQL, настроек Patroni и конфигураций ресурсов Kubernetes (например, affinity, tolerations).

## Сценарии использования

- **База данных облачного приложения**\
  Обеспечивает экземпляры PostgreSQL с высокой доступностью для архитектур микросервисов, seamlessly вписывающихся в CI/CD пайплайны.

- **Критические бизнес-системы**\
  Финансовая высокая доступность, поддерживающая переключение на резервирование уровня второго уровня и восстановление без потери данных.

- **Соответствие требованиям безопасности данных**\
  Политики хранения резервных копий и контроль доступа, соответствующие регуляторным требованиям, таким как GDPR.

- **Среды разработки и тестирования**\
  Быстрое клонирование продакшн-баз данных и поддержка создания изолированных тестовых сред по запросу.

## Ограничения

- **Поддержка версий**\
  В настоящее время поддерживаются версии PostgreSQL 11, 12, 14.

- **Размер кластера**\
  Один кластер поддерживает до 15 реплик.

- **Требования к хранилищу**\
  Необходимо использовать классы хранилищ, которые поддерживают динамическое выделение ресурсов.

- **Версия Kubernetes**\
  Требует версию Kubernetes 1.19 или выше.

- **Требования к ресурсам**: Минимальная рекомендованная конфигурация для узла — 2 процессорных ядра и 4 ГБ памяти. Фактические требования зависят от нагрузки и размера кластера.

## Преимущества поддержки PostgreSQL от Alauda

Поддержка PostgreSQL от Alauda упрощает сложность запуска и управления PostgreSQL на Kubernetes. Используя API, встроенные в Kubernetes, и автоматизацию, она предлагает:

- **Упрощенное управление**: Уменьшает операционные затраты за счет декларативной конфигурации и автоматизированного управления жизненным циклом.
- **Готовность к производству**: Предоставляет необходимые функции, такие как высокая доступность, резервное копирование/восстановление и безопасность, которые необходимы для рабочих нагрузок производства.
- **Интеграция с облачными решениями**: С seamlessly интегрируется с инструментами экосистемы Kubernetes для мониторинга, ведения журналов и хранения данных.
- **Гибкость**: Поддерживает различные версии PostgreSQL и предлагает обширные параметры конфигурации для адаптации развертываний под конкретные нужды.

Будь вы в процессе миграции существующих баз данных PostgreSQL на Kubernetes или развертывания новых облачных приложений, поддержка PostgreSQL от Alauda предоставляет надежное, масштабируемое и эффективное решение.
