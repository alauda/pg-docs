# 连接池

## 功能介绍

pgBouncer 提供连接池功能，主要特性包括：

1. **连接复用**：减少数据库连接开销
2. **负载均衡**：自动分发查询到不同节点
3. **连接限制**：防止单个客户端占用过多连接
4. **查询缓存**：提高重复查询性能

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| max_client_conn | 100 | 最大客户端连接数 |
| default_pool_size | 20 | 默认连接池大小 |
| reserve_pool_size | 5 | 预留连接池大小 |
| pool_mode | session | 连接池模式 |
| auth_type | md5 | 认证方式 |

## 使用场景

1. 高并发应用
2. 微服务架构
3. 读写分离场景
4. 连接数限制场景

## 注意事项

1. 合理设置连接池大小
2. 监控连接池使用情况
3. 定期优化查询
4. 配置合理的超时时间
