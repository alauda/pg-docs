# 介绍

## PostgreSQL Operator 介绍

Alauda Database for PostgreSQL 是基于 Zalando Postgres Operator 构建的 Kubernetes 原生 PostgreSQL 管理解决方案。它通过自定义资源定义（CRD）实现 PostgreSQL 集群的全生命周期管理，支持自动故障转移、滚动升级、备份恢复等核心运维能力，帮助企业在 Kubernetes 上轻松运行生产级 PostgreSQL 数据库。

## 核心优势

PostgreSQL Operator 的核心优势如下：

- **自动化运维**  
  基于 Operator 模式自动处理集群部署、扩缩容、备份恢复等复杂操作，减少人工干预

- **高可用架构**  
  集成 Patroni 实现自动故障转移，确保数据库服务持续可用

- **弹性扩展**  
  支持在线调整 CPU、内存、存储等资源，实时响应业务需求变化

- **多云就绪**  
  提供与 AWS S3、Google Cloud Storage、Azure Blob Storage 的无缝集成，实现跨云备份恢复

- **安全合规**  
  支持 TLS 加密通信、RBAC 权限控制、定期密码轮转等企业级安全特性

- **可观测性**  
  内置 Prometheus 指标监控和日志集成，提供完整的数据库运行洞察

## 应用场景

PostgreSQL Operator 的主要应用场景包括：

- **云原生应用数据库**  
  为微服务架构提供高可用 PostgreSQL 实例，完美契合 CI/CD 流水线

- **混合云数据服务**  
  通过统一的 Operator 管理跨云/跨集群的 PostgreSQL 实例

- **关键业务系统**  
  金融级高可用保障，支持秒级故障切换和零数据丢失恢复

- **数据安全合规**  
  满足 GDPR 等法规要求的备份保留策略和访问控制

- **开发测试环境**  
  快速克隆生产数据库，支持按需创建隔离的测试环境

## 使用限制

- **版本支持**  
  当前主要支持 PostgreSQL 11,12,14 版本

- **集群规模**  
  单个集群最多支持 15 个副本节点

- **存储要求**  
  必须使用支持动态供给的存储类（StorageClass）

- **Kubernetes 版本**  
  需要 Kubernetes 1.19 及以上版本

- **资源要求**  
  最小配置要求 2 CPU 核/4GB 内存
